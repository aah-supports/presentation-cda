### Glossaire des Termes de la CDA

1. **API (Application Programming Interface)**  
   Interface permettant à deux applications de communiquer entre elles en utilisant un ensemble de règles définies (comme des requêtes HTTP, des méthodes et des structures de données).

2. **Back-end**  
   Partie d'une application qui gère la logique métier, l'accès aux données, et l'interaction avec les utilisateurs via une interface (souvent invisible pour les utilisateurs).

3. **Base de Données (BDD)**  
   Système permettant de stocker, organiser, et récupérer des données de manière structurée, généralement géré par un SGBD (Système de Gestion de Base de Données).

4. **CRUD (Create, Read, Update, Delete)**  
   Les quatre opérations de base pour manipuler des données dans une base de données : créer, lire, mettre à jour et supprimer.

5. **DevOps**  
   Méthodologie de développement qui combine développement et opérations pour améliorer la collaboration, l'automatisation et la gestion des processus de déploiement.

6. **Front-end**  
   Partie d'une application qui interagit directement avec l'utilisateur, généralement développée avec des technologies comme HTML, CSS et JavaScript.

7. **Framework**  
   Un ensemble d'outils, de bibliothèques et de conventions de programmation qui facilite le développement d'applications en fournissant des structures réutilisables.

8. **Git**  
   Système de gestion de version décentralisé permettant aux développeurs de suivre les modifications apportées au code source et de collaborer efficacement sur des projets.

9. **GitHub / GitLab / Bitbucket**  
   Plateformes basées sur Git qui permettent d'héberger des projets et de collaborer en ligne.

10. **IDE (Integrated Development Environment)**  
    Environnement de développement logiciel qui intègre un éditeur de code, un compilateur, un débogueur, etc., pour faciliter la programmation.

11. **JavaScript (JS)**  
    Langage de programmation principalement utilisé pour le développement web côté client (navigateur). Il permet d'ajouter des fonctionnalités interactives aux pages web.

12. **JSON (JavaScript Object Notation)**  
    Format léger de données, principalement utilisé pour l'échange d'informations entre un client et un serveur via une API.

13. **JWT (JSON Web Token)**  
    Format sécurisé pour la transmission d'informations entre parties sous forme d'objet JSON, souvent utilisé pour l'authentification et l'autorisation.

14. **MVC (Model-View-Controller)**  
    Patrons de conception qui sépare l'application en trois composants : le modèle (données), la vue (interface utilisateur) et le contrôleur (logique de traitement).

15. **NoSQL**  
    Type de base de données qui ne suit pas le modèle relationnel traditionnel. Elle est plus flexible et adaptée aux applications avec de grandes quantités de données non structurées ou semi-structurées.

16. **OOP (Object-Oriented Programming)**  
    Paradigme de programmation basé sur la création d'objets, qui sont des instances de classes. Ce paradigme met l'accent sur la modélisation d'entités du monde réel.

17. **PHP**  
    Langage de programmation côté serveur utilisé principalement pour le développement d'applications web dynamiques.

18. **REST (Representational State Transfer)**  
    Architecture pour la création de services web qui repose sur des principes comme l'utilisation des verbes HTTP (GET, POST, PUT, DELETE) et des URI pour l'accès aux ressources.

19. **SGBD (Système de Gestion de Base de Données)**  
    Logiciel permettant de créer, gérer et interagir avec des bases de données. Exemples : MySQL, PostgreSQL, MongoDB.

20. **Scrum**  
    Méthodologie agile utilisée pour la gestion de projets, axée sur des sprints courts (généralement 2-4 semaines) pour livrer des incréments du produit.

21. **SQL (Structured Query Language)**  
    Langage utilisé pour interroger et manipuler des bases de données relationnelles.

22. **TDD (Test-Driven Development)**  
    Méthode de développement où les tests sont écrits avant le code. Cela permet de s'assurer que le code fonctionne comme prévu.

23. **UI (User Interface)**  
    Interface utilisateur, la partie de l'application avec laquelle l'utilisateur interagit, comprenant les éléments visuels, les boutons, les menus, etc.

24. **UX (User Experience)**  
    Expérience de l'utilisateur, qui se concentre sur la facilité d'utilisation, l'efficacité et la satisfaction de l'utilisateur lors de l'interaction avec l'application.

25. **VCS (Version Control System)**  
    Système permettant de gérer les versions d'un projet. Git est un exemple de VCS.

26. **WebSocket**  
    Protocole de communication permettant d'établir une connexion bidirectionnelle persistante entre un client et un serveur, utilisé pour les applications en temps réel.

27. **Web API**  
    Interface de programmation d'application accessible via le web, permettant de communiquer avec un serveur à travers des requêtes HTTP.

28. **XSS (Cross-Site Scripting)**  
    Type d'attaque de sécurité où des scripts malveillants sont injectés dans une application web, affectant l'utilisateur final.

29. **CI/CD (Continuous Integration / Continuous Deployment)**  
    Pratiques permettant d'automatiser l'intégration continue et le déploiement continu pour rendre plus rapide et fiable le processus de livraison du code.

30. **Cloud Computing**  
    Modèle de livraison des services informatiques (serveurs, bases de données, stockage, etc.) via internet, permettant d'éviter l'achat de matériel et de logiciels coûteux.

31. **Microservices**  
    Architecture logicielle qui décompose une application en petits services indépendants, chacun ayant une fonction spécifique et communiquant via des API.

32. **Containerisation (Docker)**  
    Technique permettant de créer des environnements d'exécution isolés pour les applications, garantissant leur portabilité entre les différents systèmes.

33. **Sass / SCSS**  
    Préprocesseurs CSS permettant de faciliter la gestion des feuilles de style en offrant des fonctionnalités comme les variables, les fonctions et l'imbrication.

34. **Git Flow**  
    Workflow de gestion des branches Git qui définit des règles pour le développement de nouvelles fonctionnalités, le déploiement et la gestion des versions.

35. **Bash**  
    Interpréteur de commandes Unix, souvent utilisé dans des scripts pour automatiser les tâches sur les systèmes Linux ou macOS.

Bien sûr ! Voici d'autres termes que vous pouvez ajouter à votre glossaire pour enrichir la compréhension des étudiants sur les aspects techniques du développement d'applications :

36. **Agile**  
    Méthodologie de gestion de projet basée sur des cycles de développement courts (sprints) et une collaboration continue entre les équipes de développement et les parties prenantes.

37. **Async/Await**  
    Mécanismes en JavaScript (et dans d'autres langages) permettant de travailler avec des opérations asynchrones de manière plus lisible et plus structurée, en évitant les "callbacks" imbriqués.

42. **CORS (Cross-Origin Resource Sharing)**  
    Mécanisme de sécurité qui permet ou bloque l'accès aux ressources d'une application depuis un domaine différent de celui du serveur d'origine.

43. **Dockerfile**  
    Fichier de configuration utilisé pour définir une image Docker, qui inclut des instructions sur la façon de construire un conteneur.

44. **E2E Testing (End-to-End Testing)**  
    Tests qui simulent le comportement d'un utilisateur afin de vérifier qu'une application fonctionne comme prévu du début à la fin, en couvrant tous les composants (interface utilisateur, back-end, base de données).

45. **ES6 (ECMAScript 6)**  
    Version de JavaScript qui introduit de nouvelles fonctionnalités, telles que les classes, les modules, les promesses, et les flèches, qui rendent le langage plus moderne et plus structuré.

46. **Flask**  
    Framework léger pour Python, utilisé principalement pour créer des applications web et des API RESTful.

47. **GraphQL**  
    Langage de requête pour les API qui permet de récupérer des données exactement comme souhaité, contrairement à REST où les données sont souvent retournées sous forme de lots.

48. **Handler**  
    Fonction qui gère un événement ou une requête dans une application (par exemple, un gestionnaire d'événements dans JavaScript ou un gestionnaire de routes dans un serveur Express).

49. **Heap**  
    Zone de mémoire dans laquelle les objets sont alloués dynamiquement. Contrairement à la pile (stack), le heap permet de gérer des objets dont la taille n'est pas connue à l'avance.

50. **Heroku**  
    Plateforme de cloud computing qui permet de déployer, gérer et faire évoluer des applications web sans se soucier de l'infrastructure sous-jacente.

51. **In-memory Database**  
    Base de données qui stocke toutes ses données dans la mémoire vive (RAM) plutôt que sur des disques durs, offrant ainsi des performances accrues pour des applications nécessitant une faible latence.

52. **Jest**  
    Framework de test JavaScript souvent utilisé avec React pour tester des applications web en simulant le rendu des composants et en vérifiant leur comportement.

53. **Kubernetes**  
    Système de gestion de conteneurs qui permet de déployer, gérer et faire évoluer des applications conteneurisées dans un environnement distribué.

54. **Liskov Substitution Principle (LSP)**  
    Principe de conception objet: tout code qui fonctionne avec une classe de base doit continuer a fonctionner avec ses sous-classes. Exemple classique: si une fonction attend un `Rectangle`, lui passer un `Square` ne doit pas changer le resultat attendu.

55. **Load Balancer**  
    Dispositif ou technique permettant de répartir la charge de travail entre plusieurs serveurs ou ressources pour améliorer la performance, la disponibilité et la tolérance aux pannes d'une application.

56. **MongoDB**  
    Base de données NoSQL orientée documents qui utilise un format de données flexible basé sur JSON pour stocker des informations.

57. **OAuth**  
    Protocole d'authentification et d'autorisation qui permet à des applications tierces d'accéder à des ressources sur un serveur au nom de l'utilisateur, sans partager ses informations d'identification.

58. **ORM (Object-Relational Mapping)**  
    Technique permettant de mapper des objets dans une base de données relationnelle. Exemple : Sequelize pour Node.js ou Hibernate pour Java.

59. **PWA (Progressive Web Application)**  
    Application web qui utilise des technologies modernes pour offrir une expérience proche de celle d'une application native, incluant des fonctionnalités comme le mode hors-ligne et les notifications push.

60. **Promise**  
    Objet en JavaScript représentant une valeur qui peut être disponible maintenant, dans le futur ou jamais. Il est utilisé pour gérer les opérations asynchrones.

61. **QA (Quality Assurance)**  
    Processus visant à garantir que le code produit par les développeurs répond aux critères de qualité, en vérifiant la performance, la sécurité et l'absence de bugs.

62. **RESTful API**  
    API qui suit les principes REST, utilisant les verbes HTTP (GET, POST, PUT, DELETE) pour effectuer des actions sur des ressources représentées sous forme de données.

63. **Sequelize**  
    ORM pour Node.js utilisé pour interagir avec des bases de données SQL (comme PostgreSQL, MySQL, etc.), en facilitant les opérations CRUD.

64. **Singleton**  
    Patrons de conception qui garantit qu'une classe n'ait qu'une seule instance et fournisse un point d'accès global à cette instance.

65. **SSR (Server-Side Rendering)**  
    Technique de rendu d'un site web sur le serveur avant de l'envoyer au client. Cela permet une meilleure indexation par les moteurs de recherche et une vitesse de chargement plus rapide.

66. **Trello**  
    Outil de gestion de projets collaboratifs basé sur des tableaux et des cartes, très utilisé dans les équipes Scrum pour organiser les tâches à accomplir.

67. **Webpack**  
    Outil de bundling (assemblage de modules) pour les applications JavaScript, permettant de regrouper et optimiser les fichiers avant leur déploiement en production.

68. **Yarn**  
    Gestionnaire de paquets JavaScript alternatif à npm, utilisé pour installer, gérer et mettre à jour les dépendances des projets.

69. **Zero Downtime Deployment**  
    Méthode de déploiement permettant de mettre à jour une application sans aucune interruption de service.
