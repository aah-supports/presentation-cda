Cycle Scrum

Refinement: méthode opérationnelle

  1. Préparer les US candidates avec contexte et critères initiaux.
  2. Clarifier avec l'équipe: questions, risques, dépendances.
  3. Découper les US trop volumineuses.
  4. Vérifier la Definition of Ready.
  5. Mettre à jour le Product Backlog avant estimation.

Cas complet de soutenance (chaîne de valeur)

  1. Besoin métier → backlog priorisé.
  2. Estimation planning poker.
  3. Planification roadmap + Gantt.
  4. Tickets + documentation.
  5. Branches Git + revues.
  6. Pipeline CI/CD + tests + Sonar.
  7. Rétro et amélioration continue.

Niveau de précision attendu dans le dossier

  • Preuves planning (initial / réel / écarts).
  • Preuves agiles (backlog, votes, consensus).
  • Preuves qualité (tests, pipeline, métriques).
  • Preuves sécurité (risque → mesure → preuve).
