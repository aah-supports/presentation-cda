Glossaire Tests

Definitions essentielles + exemples JavaScript et PHP.

Retour index

Notions de base

  • Test unitaire: teste une unite de code isolee.
  • Test d'integration: teste l'interaction entre composants.
  • Test fonctionnel: valide un besoin metier observable.
  • E2E: scenario complet utilisateur de bout en bout.
  • Assertion: verification explicite du resultat attendu.
  • Non-regression: garantit qu'une evol ancienne ne casse pas l'existant.

Doubles de test

  • Mock: objet simule avec comportement et verifications d'appels.
  • Stub: objet qui renvoie une valeur fixe predefinie.
  • Fixture: jeu de donnees stable pour tests reproductibles.
  • Flaky test: test instable (resultat aleatoire), a corriger.
  • Pyramide des tests: beaucoup d'unitaires, moins d'integration, peu de E2E.
  • Coverage: part de code executee par les tests (indicateur, pas preuve absolue).

Exemple JavaScript (Jest)

import { sum } from './sum.js';

test('sum additionne deux nombres', () => {
  expect(sum(2, 3)).toBe(5);
});

Exemple JavaScript (mock)

import { countMovies } from './movieService.js';

test('countMovies utilise le repo', async () => {
  const repoMock = { findAll: jest.fn().mockResolvedValue([{ id: 1 }, { id: 2 }]) };
  await expect(countMovies(repoMock)).resolves.toBe(2);
  expect(repoMock.findAll).toHaveBeenCalledTimes(1);
});

Exemple PHP (PHPUnit)

final class Calculator {
    public function add(int $a, int $b): int {
        return $a + $b;
    }
}

final class CalculatorTest extends TestCase {
    public function testAdd(): void {
        $calculator = new Calculator();
        $this->assertSame(5, $calculator->add(2, 3));
    }
}

Exemple PHP (service + mock)

$repo = $this->createMock(MovieRepository::class);
$repo->method('findAll')->willReturn([['id' => 1], ['id' => 2], ['id' => 3]]);

$service = new MovieService($repo);
$this->assertSame(3, $service->count());

Attendu CDA

  • Jeu d'essai: entree, attendu, obtenu.
  • Trace: test lie a une fonctionnalite du projet.
  • Lisibilite: test simple, nom explicite, comportement verifiable.
  • Limites: indiquer ce qui n'est pas teste.
