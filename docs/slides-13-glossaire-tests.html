Types de tests (une definition claire)

  • Test unitaire: verifie une unite de code isolee (fonction, methode, service), sans dependances reelles.
  • Test d'integration: verifie la collaboration entre plusieurs composants (service + repository + base, par exemple).
  • Test fonctionnel/API: verifie un comportement metier observable via une route HTTP et son contrat (status, JSON, erreurs).
  • Test E2E: verifie un parcours utilisateur complet depuis l'interface (navigation, saisie, resultat visible).
  • Test de non-regression: rejoue une suite de tests pour s'assurer qu'une modification ne casse pas l'existant.

Regle simple: unitaire = local, integration = assemblage technique, fonctionnel/E2E = valeur utilisateur.

Doubles et donnees de test

  • Stub: faux objet qui renvoie des valeurs predefinies, sans verification d'appel.
  • Mock: faux objet qui permet aussi de verifier les interactions (nombre d'appels, parametres).
  • Fake: implementation simplifiee mais executable (ex: repository en memoire).
  • Fixture: jeu de donnees stable et reutilisable par plusieurs tests.
  • Flaky test: test instable (passe/echoue aleatoirement), a corriger en priorite.
  • Pyramide des tests: majorite de tests unitaires, une couche integration, peu de E2E.

Exemple long JS - test unitaire (Jest/Vitest)

export class MovieService {
  constructor(movieRepository) {
    this.movieRepository = movieRepository;
  }

  async create(input) {
    if (!input.title || input.title.trim().length < 2) {
      throw new Error("TITLE_INVALID");
    }
    if (!Number.isInteger(input.year) || input.year < 1888) {
      throw new Error("YEAR_INVALID");
    }
    return this.movieRepository.insert({
      title: input.title.trim(),
      year: input.year,
      director: input.director?.trim() ?? null
    });
  }
}

test("create normalise et enregistre un film valide", async () => {
  const repo = { insert: vi.fn().mockResolvedValue({ id: 1, title: "Stalker" }) };
  const service = new MovieService(repo);

  const created = await service.create({ title: "  Stalker  ", year: 1979, director: "Tarkovski" });

  expect(repo.insert).toHaveBeenCalledWith({ title: "Stalker", year: 1979, director: "Tarkovski" });
  expect(created.id).toBe(1);
});

test("create rejette une annee invalide", async () => {
  const repo = { insert: vi.fn() };
  const service = new MovieService(repo);
  await expect(service.create({ title: "Stalker", year: 1200 })).rejects.toThrow("YEAR_INVALID");
  expect(repo.insert).not.toHaveBeenCalled();
});

Ce test verifie la logique metier du service sans base reelle.

Exemple long JS - test fonctionnel API

import request from "supertest";
import { createApp } from "../src/app.js";
import { InMemoryMovieRepository } from "../src/repositories/in-memory.js";

describe("POST /api/movies", () => {
  test("retourne 201 et un JSON conforme", async () => {
    const repo = new InMemoryMovieRepository();
    const app = createApp({ movieRepository: repo });

    const response = await request(app)
      .post("/api/movies")
      .send({ title: "In the Mood for Love", year: 2000, director: "Wong Kar-wai" })
      .expect(201);

    expect(response.body).toEqual({
      id: expect.any(Number),
      title: "In the Mood for Love",
      year: 2000,
      director: "Wong Kar-wai"
    });
  });

  test("retourne 400 si le payload est invalide", async () => {
    const repo = new InMemoryMovieRepository();
    const app = createApp({ movieRepository: repo });

    const response = await request(app)
      .post("/api/movies")
      .send({ title: "", year: 1700 })
      .expect(400);

    expect(response.body.error).toBe("VALIDATION_ERROR");
  });
});

Ce test verifie route + validation + contrat JSON, mais pas l'interface graphique.

Exemple long PHP - unitaire et fonctionnel

final class MovieServiceTest extends TestCase
{
    public function testCreateRejectsEmptyTitle(): void
    {
        $repo = $this->createMock(MovieRepository::class);
        $repo->expects($this->never())->method('insert');

        $service = new MovieService($repo);

        $this->expectException(InvalidArgumentException::class);
        $this->expectExceptionMessage('TITLE_INVALID');
        $service->create(['title' => '', 'year' => 2001]);
    }
}

final class MovieApiTest extends WebTestCase
{
    public function testCreateMovieReturns201(): void
    {
        $client = static::createClient();
        $client->request('POST', '/api/movies', [], [], ['CONTENT_TYPE' => 'application/json'],
            json_encode(['title' => 'Drive My Car', 'year' => 2021])
        );

        $this->assertResponseStatusCodeSame(201);
        $json = json_decode($client->getResponse()->getContent(), true);
        $this->assertSame('Drive My Car', $json['title']);
    }
}

Unitaire: regles de domaine. Fonctionnel: contrat HTTP et structure de reponse.
