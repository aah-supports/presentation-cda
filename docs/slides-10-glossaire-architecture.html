REST et API

  • REST: style d'architecture web orienté ressources et HTTP.
  • RESTful: API qui respecte les contraintes REST (URI, verbes HTTP, stateless).
  • Ressource: objet métier exposé via une URI, ex: /api/movies/12.
  • Stateless: chaque requête contient son contexte, le serveur ne garde pas de session implicite.
  • Idempotence: rejouer une action ne change pas le résultat final (cas de PUT/DELETE).
  • HATEOAS: niveau REST avancé où la réponse contient des liens d'actions possibles.

MVC et Clean

  • MVC: séparation Model (données), View (affichage), Controller (orchestration).
  • Service Layer: couche de règles métier entre contrôleur et accès données.
  • Repository: abstraction de l'accès aux données (SQL, mémoire, autre).
  • Clean Architecture: dépendances orientées vers le cœur métier.
  • Use Case: action métier applicative (ex: "ajouter un film").
  • Ports and Adapters: interfaces stables + implémentations techniques interchangeables.

Architectures Front-end

  • MPA: application multi-pages, chargement serveur à chaque navigation.
  • SPA: application page unique, navigation majoritairement côté navigateur.
  • SSR: rendu HTML côté serveur avant envoi au client.
  • CSR: rendu principal côté client via JavaScript.
  • SSG: pages générées statiquement à la build.
  • BFF: backend dédié à un front (web/mobile) pour adapter les données.

Middleware

  • Middleware: fonction intermédiaire entre requête et réponse.
  • Pipeline: chaîne de middlewares exécutés dans un ordre précis.
  • Validation Middleware: vérifie les entrées avant le contrôleur.
  • Auth Middleware: contrôle identité et droits d'accès.
  • Error Handler: centralise la gestion des erreurs et le format de réponse.
  • Rate Limiting: limite les requêtes pour protéger l'API.

Exemple simple

  • Route REST: GET /api/movies retourne une liste de ressources.
  • Controller: reçoit la requête et délègue au service métier.
  • Service: applique les règles de tri/filtrage.
  • Repository: lit les données en base.
  • Middlewares: auth + validation + gestion d'erreurs autour de la route.
