Abstract: rôle central du support

Le fichier Abstract structure la certification: livrables, méthodologie attendue, épreuve orale.

  • Le jury attend une démarche complète, pas une simple démo.
  • Sécurité et qualité doivent traverser tout le projet.
  • Le dossier projet est le document principal de certification.

3 activités du titre CDA

  • Concevoir/développer des composants d'interface sécurisés.
  • Concevoir/développer la persistance des données sécurisée.
  • Concevoir/développer une application multicouche répartie sécurisée.

Livrables attendus

Dossier professionnel (DP)

Document ministériel alimenté régulièrement avec des expériences réelles.

Dossier projet

Document principal de certification (volumétrie conséquente, démarche complète).

Dossier projet: plan type (1/2)

  • Compétences du référentiel couvertes.
  • Résumé du projet en anglais (200-250 mots).
  • Cahier des charges / expression des besoins.
  • Gestion de projet (planning, suivi, environnement humain/technique, qualité).
  • Spécifications fonctionnelles.

Dossier projet: plan type (2/2)

  • Spécifications techniques (dont sécurité).
  • Réalisations argumentées avec extraits de code significatifs.
  • Jeu d'essai d'une fonctionnalité représentative.
  • Veille sécurité menée pendant le projet.
  • Description d'une situation de recherche technique.

Gestion de projet attendue (abstract)

  1. Créer le Product Backlog priorisé.
  2. Estimer en story points (planning poker).
  3. Construire roadmap et jalons.
  4. Préparer sprint backlog (tâches + estimations).
  5. Suivre les écarts et replanifier.
  6. Conserver les preuves (board, Gantt, commits).

Spécifications fonctionnelles attendues

  • Cas d'usage / user stories.
  • Entrées / sorties attendues.
  • Critères d'acceptation.
  • Gestion des erreurs côté métier.
  • Matrice de traçabilité exigence → US → test.

Spécifications techniques attendues

  • Périmètre technique et stack.
  • Architecture et flux principaux.
  • Modèle de données, API et erreurs.
  • Sécurité: validation, secrets, anti-injection, traces.
  • Qualité, performance, déploiement, traçabilité.

Exemples d'outils / stacks (non imposés)

Gestion projet

Jira, Trello, Notion, Confluence, Gantt.

Développement

Node/TypeScript, Symfony/PHP, Java/Spring, .NET.

Données

PostgreSQL, MySQL, MongoDB, Redis.

Tests & qualité

Jest, PHPUnit, Cypress, SonarQube.

CI/CD

GitHub Actions, GitLab CI, Jenkins, Docker.

Monitoring

ELK/Kibana, Grafana/Loki, Sentry.

Le jury n'impose pas une stack unique: il évalue la cohérence des choix, la maîtrise et les preuves.

Jeu d'essai: format attendu

Données en entrée

Payload/API ou actions UI.

Données attendues

Résultat théorique prévu.

Données obtenues

Résultat réel observé + analyse des écarts.

Vocabulaire: test fonctionnel/E2E, test intégration API, test unitaire.

Veille sécurité et recherche technique

Veille sécurité

Objectif, périmètre, méthode, journal daté, actions correctives, bilan.

Situation de recherche

Contexte, blocage, démarche, options, mise en œuvre, résultat, capitalisation.

Projet perso (fil rouge)

  • Initiative personnelle démarrée tôt dans l'année.
  • Traité comme une demande professionnelle complète.
  • Permet de combler des manques éventuels côté entreprise.
  • Renforce l'employabilité en fin de formation.

Épreuve orale: structure

40 min

Présentation projet (incluant résumé anglais).

45 min

Entretien technique avec le jury.

20 min

Entretien final (posture et enjeux métier).

Canevas attendu en présentation projet

  • Entreprise/service et contexte du projet.
  • Contraintes, livrables et analyse du besoin.
  • Gestion de projet et qualité.
  • Conception, codage, interface et jeu d'essai.
  • Exemple de recherche depuis source anglophone.
  • Synthèse: difficultés et satisfactions.

Référentiel des compétences

Compétences pro

Env. de travail, interfaces, composants métier, contribution projet.

Compétences transversales

Communication, résolution de problème, apprentissage continu.

Activités spécifiques

Application sécurisée en couches, préparation du déploiement.

Normes

ANSSI, RGPD, accessibilité, écoconception.

Compétences principales détaillées

  • Conception: spécifications claires + architecture + modèle de données.
  • Développement: cycle complet du code à la mise en prod.
  • BDD: SQL/NoSQL, requêtes efficaces, intégrité.
  • Tests: unitaires, intégration, fonctionnels, validation avant release.

Savoir-faire techniques détaillés

  • Programmation multi-langages selon contexte projet.
  • Architecture systèmes, performances et scalabilité.
  • CI/CD: Git, Jenkins, Docker, automatisation.
  • API REST/GraphQL, sécurité et gestion d'erreurs.

Savoir-faire organisationnels

  • Agile (Scrum/Kanban), planification et priorisation.
  • Gestion de projet et suivi des obstacles.
  • Communication claire avec clients/PM/équipes.
