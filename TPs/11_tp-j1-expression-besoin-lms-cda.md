# TP 11 - Jour 1 : Expression du besoin (scope CDC)

Durée conseillée : 1 journée

Objectif : produire uniquement le **chapitre CDC** (cahier des charges fonctionnel) d'un mini projet.

Contexte fil rouge : application web "Films d'art et essai" pour passionnés de cinéma.

## 1. Règle de scope (obligatoire)

Ce TP est un exercice de **cadrage fonctionnel**.

Vous restez dans le périmètre **expression du besoin / CDC**.

Ce qui est demandé :

- contexte et objectif du projet
- cible et valeur attendue
- périmètre MVP / hors périmètre
- contraintes
- acteurs et responsabilités
- parcours utilisateur simplifié
- hypothèses, risques et questions ouvertes

Ce qui n'est pas demandé dans ce TP :

- backlog détaillé / user stories sprintables
- planification de sprint
- architecture technique
- schémas BDD ou API
- conception UML détaillée

## 2. Lien avec le dossier CDA

Dans ce TP, vous rédigez un extrait directement réutilisable dans le dossier CDA :

- section **Expression du besoin et périmètre**
- section **Contexte de départ**
- section **Objectifs mesurables**
- section **Périmètre**
- section **Contraintes**

Référence : `annexes/DossierCDA.md`

## 3. Production attendue en fin de J1

1. Contexte de départ (problème, cible, valeur)
2. Objectifs mesurables (2 à 3 objectifs)
3. Périmètre MVP + hors périmètre
4. Contraintes (délai, technique, ressources)
5. Acteurs et responsabilités
6. Parcours utilisateur principal (5 étapes max)
7. Questions ouvertes / risques / hypothèses

## 4. Plan de travail conseillé (1 journée)

### Bloc 1 (45 min)

- Reformuler le besoin en langage métier.
- Identifier la valeur attendue.

Sortie : contexte + objectif global.

### Bloc 2 (1h)

- Définir le périmètre MVP.
- Définir explicitement le hors périmètre.

Sortie : périmètre cadré.

### Bloc 3 (1h)

- Identifier acteurs et responsabilités.
- Rédiger un parcours utilisateur simple.

Sortie : acteurs + parcours.

### Bloc 4 (1h)

- Recenser contraintes, hypothèses, risques.
- Lister les questions ouvertes.

Sortie : registre de points de vigilance.

### Bloc 5 (1h15)

- Mise en forme finale du chapitre CDC.
- Relecture croisée équipe.

Sortie : rendu final J1 prêt dossier.

## 5. Critères d'évaluation

- Pertinence du cadrage CDC : 40%
- Clarté du périmètre (MVP vs hors périmètre) : 30%
- Cohérence des acteurs/parcours : 20%
- Qualité des risques/questions ouvertes : 10%
